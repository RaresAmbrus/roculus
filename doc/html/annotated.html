Class List
Here are the classes, structs, unions and interfaces with brief descriptions:
BaseApplicationOgre BaseApplication class The BaseApplication class is the central class in the project. It brings all systems together: *initialization and control of the rendering process *handling keyboard/mouse input *connection to ROS for joystick input, image sensor data, waypoint data, etc. *cleaning up after the application finished
DoorRepresents a Door in the demo-game. A door is a GameObject and takes care of the appearance, as well as the behaviour of a door
GameSingleton pattern class to manage the demo-game. This class manages the demo-game and is responsible for all related objects, their (repeatable) initialization and the game process
GameCFGParserParses the 'game.cfg' file. This class provides an easy-to-use parser for the config file of a game. Note that the filename is hardcoded in the constructor
GameObjectDescribes a generic game object. This class is an abstract description of a game object. It give some default functionality and is otherwise used to inherit the properties to its child classes: Door, Key, Treasure and Lock
GlobalMapUtility class for the 2D map. This class handles the 2D map in the scene
KeyRepresents a Key in a Game. This class is a more detailed description of the GameObject Key. It specifies how to display and what to do with a key
LockDescribes a Lock in a Game. This class specifies the details to display a lock and implements its behaviour
OculusHandler for the Oculus Rift using SDK1. Written by Kojack 2013 (C). Slightly modified to take the PlaberBody scene node as parent to the camera setup
PlayerBodyHandles the PlayerBody motion. Extension class to handle the player motion independent from the Oculus Rift head motion. The Oculus motion happens on top of this, which is achieved by making the PlayerBody's scene node the parent for the Oculus handler. Since this was one of my first implementations (very unclean) this step, in fact, happens in BaseApplication
RobotRepresents a Robot. This class handles the robot avatar
RoculusClass to take care of the scene and object creation. This class fills the scene with objects and in most cases creates the geometries manually. It has a method to parse room recordings in folders in order to include them into the scene. Though it is the nominal main class, the majority of the implementation is found in the BaseApplication class
RoomDescribes a Room in a Game. This class collects all parameters for a Room. It keeps track of the corresponding WayPoints, Door nodes and event nodes and provides some utility functions
SnapshotCollects all data that describes a Snapshot and provides a method to insert it into the scene. (Not the smartest implementation, should probably be constructed given the scene manager and do the object creation by itself. Static counter would help to identify total number of Snapshots to link to individual materials, textures etc.)
SnapshotLibraryGroups multiple Snapshots in a library (vector). Furthermore, this class manages the memory and preallocates Ogre::Textures, Materials and SceneNodes, whenever needed
TreasureDescribes the appearance and behaviour of a Treasure in the Game. This class is similar to the other GameObjects
Video3DHandles the 3D video stream. Similar to a Snapshot, but uses DYNAMIC and DISCARDABLE textures instead, which are updated, not placed. (Not the smartest implementation, should probably be inherit properties from Snapshot...)
WayPointDefines a WayPoint for the Game. Waypoints are possible navigation targets and are used to place GameObjects, like Keys and Locks. The list of WayPoints is held by the Game class, but they are read from other classes quite often, usually passed as a pointer or reference. A WayPoint can be made inaccessible for the selection by the player setting the accessible flag
